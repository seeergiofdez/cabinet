Hi,
In order to reduce GitLab's Usage Quotas of bbjprojek and ragyxo groups, it's encouraged to use one of the following recommendations:

Skipping CI when possible
You don't need to build a site or deploy a release everytime commits are pushed. Although this is the default behaviour for most of the repos (see this as an example), it's not always necessary.
You can add "[ci skip]" or "[skip ci]" on our last commit to be pushed, and GitLab will skip the pipeline for that push.

Note that you always can manually generate a pipeline if you need, not needing to commit and push another changeset. See this other example.

Set to manually build
You can also disable automatic pipelines on some projects that doesn't require to build periodically. To do so, go to repo settings, and go to pipelines configurator. You can set and remove rules for the pipeline.

All these recommendatiosn will guarantee a better organization operation and reduce carbon footprint and become more sustainable.
